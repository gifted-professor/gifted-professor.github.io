浏览器工作原理

当我们输入一串 url 的时候，浏览器首先会进行 DNS 解析，将其转换为一串 ip 地址，之后浏览器会根据这一串 ip 地址找到这台服务器，从而接收到一个 html 文件，至于一些 css，js 等资源，会随着浏览器解析 html 文件的时候逐步向服务器发送请求，下载下来

1⃣️ 浏览器内核

主要知道有一个 Blink 就可以啦

其实浏览器引擎就是 排版引擎(layout engine)、页面渲染引擎…..

2⃣️Js 引擎

高级编程语言都是需要转化为最终的机器指令执行的 javascript 引擎 其实就是将 javaScript 代码翻译成 CPU 指令（0100101001001）

Js 引擎其实分为 WebCore 和 JavaScriptCore 前者用于 HTML 解析、布局、渲染等工作，后者负责解析、执行 js 代码

3⃣️V8 引擎的架构

  • 什么是 V8 引擎？？？
    • v8 引擎其实就是一个 JaveScript 引擎
    • v8 引擎由 C++ 编写 目前用于 Chrome 和 Node.js
    • V8 引擎的作用如下

然后就是语法分析 将代码转化为 AST 抽象语法树 转化为 抽象语法树 的原因是 抽象语法树的代码是树状结构，并且抽象语法树中的关键字较为固定 这样有助于 v8 引擎对于 js 代码的解析

例如 babal 的工作原理就是 ts code ==> AST ==> new AST ==> generate code ==> js code

vue template: template ==> AST ==> document.createVnode ==> js code

  • **iginition **

Ignition是一个解释器，可以将 AST 抽象语法树转换为 bytecode

将代码先转换为字节码之后再转换为机器码的原因是 js 代码可能会跑在不同的系统中 -> macod window linux，而在不同的环境中就会有不同的CPU，不同的CPU的架构是不同的，执行相同操作的指令是不同的，而字节码是跨平台的，不管是在什么平台，最终都可以转化为 汇编指令 最后在不同平台下生成适配的 机器指令，最后运行

  • TurboFan 是一个编译器，可以将字节码编译为 cpu 可以直接执行的机器码

由于iginition库对于 字节码 的处理事先转化为 汇编指令， 然后再转化为 机器指令，如果每次的解析过程都是这样子的话，对于一些高频函数的解析就会变的低效 所以就出现了 TurboFan

TurboFan 库对于 js 代码的处理是对一些高频代码（函数）进行分析，如果发现出现频率高的话就会将其标记一个hot（热函数，也就是代表这个函数可能会被多次执行），这样的话就可以将 AST 直接解析成 MaChineCode 优化的机器码 下次就可以直接使用优化后的机器码，可以大大提升效率

  • Deoptimization

这个其实可以理解为是取消优化的意思，也就是将 优化后的机器码 MachineCode 重新转化为 byteCode

为什么要有这个操作呢？

Deoptimezation的作用就是对这种情况进行检测，同时对代码进行优化，如果监测到这种情况的话就会重新将 MachineCode 转化为 bytecode 然后变成汇编指令，最后再变成 机器指令

所以这也是 typeScript 出生的原因，由于有了类型限制，V8 引擎对于 js 代码的解析会效率更高， 《累死自己，爽死机器》~~~

浏览器内核将源码交给 v8 引擎， Stream 获取到代码之后会进行编码转换…..
