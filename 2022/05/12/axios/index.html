🍔关于Qs

qs是一个npm管理的包 可以通过 npm i qs 来安装

基本上用到的命令就是 qs.parse() qs.stringify()

  1. qs.parse 将 URL 解析成对象的形式

  1. qs.stringify()将对象 序列化成URL的形式 以 & 进行拼接

  1. JSON.stringify() 跟 qs.stringify()

最主要的区别就是一个是将字符串转换成了对象的形式 一个就是将字符串转换成 URL 的形式

关于Qs的其他东西就不多说了 了解一下就好了 毕竟只是想知道这个东西 具体可以看看这个

🍔关于.env

这个东西在之前实战 Vue3.2 的时候就有接触到了 也是一知半解 趁着现在了解一下吧

文件格式

  • .env 全局默认配置文件，不论什么环境都会加载合并
  • .env.development 开发环境下的配置文件
    • npm run serve(或者npm run dev 主要看 package.json) 会合并 .env 文件
  • .env.production 生产环境下的配置文件
    • npm run build 会合并 .env 文件

其他的请求文件

关于 data 跟 params 的区别

params：使用params作为参数时 参数会直接追加到 url 的请求字符串中 一般用于GET请求 然后有测试过如果使用 params 的POST请求 也是可以成功的

data：使用 data 的话 参数是添加到请求体 body 里面 一般用于POST请求
