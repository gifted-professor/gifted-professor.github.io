🍔什么是 Node ？

Node.js简单来说其实就是一个可以让 JS 运行在服务端的环境

🍔Js运行在浏览器跟Node上的区别

Js代码在浏览器运行的时候 就像是进入了一个封闭的环境 不能访问本地电脑上的一切资源 一定要访问的话就需要使用浏览器提供的 API 还需要用户授权 功能上相对 哒咩

而 JS 代码运行在 Node 上就可以做很多事情了 本地文件读取 连接数据库存储数据 发送短信 邮箱……

🍔关于返回的状态码

有点常用的 主要就是在请求的时候经常会报错 所以还是得看一下

点我查看状态码大全

HTTP 状态码由三个十进制数字组成，第一个十进制数字定义了状态码的类型。响应分为五类：

信息响应(100–199)，成功响应(200–299)，重定向(300–399)，客户端错误(400–499), 服务器错误 (500–599)

常见状态码👇

状态码 状态描述
100 继续 客户端应继续请求
200 请求成功 一般用于用于 POST 和 GET 请求
201 已创建。成功请求并创建了新的资源
202 已接受。已接受请求 但是未处理完成
203 非授权信息。请求成功 但是请求的meta数据不在原始的服务器
300 - 399 重定向。大概一起就原本请求的链接地址已经改变 大概就是这个意思吧
400 客户端请求的语法错误 服务端无法理解
401 请求要求用户的身份认证
403 服务端理解请求 但是拒绝执行此请求
404 服务端无法根据客户端的请求找到资源 “经典 404 ”
500 服务器内部错误 无法完成请求
501 服务器请求的功能 无法完成请求
505 服务器不支持的HTTP协议的版本 无法完成处理

  • 登录: /api/login
