后台管理系统总结

使用vue ui创建一个项目

  • 首先可以将路径设置在自己想存放文件夹的位置

  • 手动配置 一般就是选择 **Router Vuex Eslint就算了 **

  • 记得不保存预设

  • 点击依赖 添加一个**axios**

  • 这里使用的是Vue3.2的版本 所以vue-router还有vuex的版本也是是需要限定的 防止出现冲突或者什么版本错误 =======>>>> npm i vue@3.2.8 vue-router@4.0.11 vuex@4.0.2

  • 这个时候需要重新 npm i 一下 可以减少报错 至于是为什么我也不理解

代码格式化

  • 首先先安装一下 Prettier - xxxxxxx

  • 然后点击设置按钮 点击设置 搜索save

  • 勾选 Format On Save

  • 在随便一个主页先右键点击 选择 使用…格式化文档

  • 配置默认格式化程序 然后选中 Prettier

  • 关于 **.prettierrc**文件 可以在 D:\项目文件\web前端文件\后台管理1.0 母本 防出事 中查找

  • 然后关于ESlint的 我实在是不想搞 但是还是写一些吧 这个是为了解决 **Prettier跟Eslint**之间的冲突

    在rules配置项中加入以下👇 当然 这个问题要是直接去那个网上搜肯定也是有的

关于 git commit 的规范

首先安装 commitizen和 cz-customizable 自己在网上找吧 到时候可能版本什么的又会变了

package.json中新增配置

这里面需要到的 cz-customizable 也是在 D:\项目文件\web前端文件\后台管理1.0 母本 防出事 中查找

这样的话就可以只用git cz来代替git commit了

这样的话相对于直接使用 git commit -m "feat: xxx" 的话方便了很多 因为 **git cz**的话可以直接选择而不用手写

强制git代码提交规范

  • npm i --save-dev @commitlint/config-conventional@12.1.4 @commitlint/cli@12.1.4
  • 这个是我看的时候的一些版本 等到到时候写的时候可能又会不一样了 所以版本号的话还是先搜一下 确定没问题了再搞吧
  • npm i husky@7.0.1 --save-dev //这里也是 版本问题的话 到时候可以自己去看看
  • npx husky i //这个就是husky的初始化
  • 这里面需要到的 commitlint.config.js 也是在 D:\项目文件\web前端文件\后台管理1.0 母本 防出事 中查找
  • 然后需要在 **package.json**中的 script 配置项中加入 "prepare":"husky install" 这样的话就可以使用 npm run prepare 命令了 这个时候会生成一个 .husky文件夹
  • 执行 npx husky add .husky/commit-msg 这个时候就会多出一个commit-msg文件
  • 在commit文件中写入这段话 npx --no-install commitlint --edit

然后错误的commit提交格式就会报错 无法进行commit

强制代码规范

  • npx husky add .husky/pre-commit

  • 在多出来的一个文件中写入 npx lint-staged

  • 使用这段话的话需要在package.json 中进行配置

按需引入elementplus

  • 首先安装一个element
  • 然后在快速开始那里找到按需导入的指令
  • 由于使用的是Webpack 需要往下翻 找到需要的 Webpack 配置 cv一下
  • 创建 vue.config.js 将上面那段话放进来 这个时候再执行 npm run serve 可能会报错 就是说没有找到 **plugins**这个东西 这个时候就要小改一下

Vue3中在js里面写css代码

可以在main.js里面引入一些scss文件 这个东西也是在 母本链接里面

关于引入图标文件…

因为是按需引入 所以一般我们都是通过import的方式来引入什么东西 只要记住一个 从哪里安装就是从那里引入 这个就好了 一般安装就是 npm i @xxxxxxx 那我们引入的话就是 **import { xxx } from '@xxxxx'**就好了
