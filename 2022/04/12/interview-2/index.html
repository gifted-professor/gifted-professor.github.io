面试题2 - 闭包

正文👇

JS中关于闭包的定义

…….

支持头等函数的编程语言。。。。。

必要在实现上是一个结构体 他存储了一个函数换关联的环境（相当于一个符号查找表） 后面就知道 其实这个意思就是 👇

MDN 中对于必要的解释就是 ===> 一个函数和对其周围状态的引用捆绑在一起的时候 这样的组合就叫做必要

也就是说 必要可以让你在内层函数中访问到外层函数的作用域（这个很好理解了 就差不多是我上面说的那个例子 在父级作用域中找到了自己需要的变量）

而且在 JS 中 每当你创建出一个函数 闭包就会被创建出来

  • 简单的理解

一个普通的函数 如果它可以访问外层作用于的自由变量 那么这个函数就是一个闭包

广义的角度说 js 中的函数都是闭包

侠义的角度上来说 js 中的一个函数 如果访问了外层作用域的变量 那么它是一个闭包

函数的执行

null 其实指向的内存地址是0x0

函数的执行过程

首先会创建一个 GO 对象

会有一个函数执行调用栈 在函数编译的时候就会先创建一个全局执行上下文

执行上下文都会有一个VO 全局执行上下文中的 VO 就是对应的 GO

GO 里面会有函数中的各种变量和参数 遇到 函数的时候就会生成一个函数对象 会给他分配一个地址

函数对象里面有两个参数 一个是 父级作用域（parentScope） 还有会有个函数的执行体 其中父级作用域的 value 就指向了 GO 而 GO 对象就会有一个地址

当函数开始执行的时候 就会在调用栈中在创建出来一个函数执行上下文 这里面还是有一个 VO VO指向的是AO 但是此时没有AO 所以就会创建出一个AO对象 AO对象对应的是一个地址 里面存放着的是foo函数中的变量 变量刚开始的 value 都是 undefined 在函数执行之后会开始赋值 当函数执行完毕之后 函数执行上下文也会被销毁 随之里面的AO对象也会被销毁

函数对象是基于 在AO对象中出现了函数 就会生成一个函数对象

🍔闭包实际运用场景

其实这个之前就应该记的 只是突然忘了 刚好今天看到一个文章就写一下

  • 防抖节流
    可能刚看到这个会有一点疑惑 什么是防抖节流 举个简单的例子 就是在你进行搜索的时候 当你一直在输入的时候 搜索出来的数据不会实时的改变 会有一段间隔时间等待输入完成才实现 这样是为了防止一直请求数据导致不必要的资源浪费
  • 闭包可以用在对象中创建自由变量
  • 必要可以在JavaScript种模拟块级作用域

🍔小知识点

  • 内存泄漏

关于内存泄漏 其实就是在函数执行完毕之后 因为还有引用指向这个函数 导致这个函数不能被销毁 解决方案就是给指向它的这个函数赋值为 null 这样的话就会让这个函数销毁（Ps：这个长得好像有点像操作系统中的僵尸进程 就是函数执行完毕 但是还占用 CPU PCB 什么的东西 不多说）

  • AO对象

关于AO对象 其实就是 Activation Object 叫做活动型对象 这个听起来可能不是很熟悉 那就换一个词 叫做函数执行上下文

  • 浏览器中的进程

好像浏览器中每个网页就代表了一个进程

  • JS中的 pop shift push

这三个函数分别就是对数组做一些操作

pop是抛出该数组的最后一项数据

shift是删除该数组的第一个数据

push是往数组中添加数据
